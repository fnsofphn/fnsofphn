# Codex Optimization - Optimized

## 1. Context Discipline
- Moi tin nhan moi lam assistant phai xu ly lai lich su hoi thoai; thread cang dai cang ton token.
- Khi thread dai, yeu cau tom tat trang thai, mo thread moi, dan ban tom tat vao de tiep tuc.
- Khong de context gan day; reset som khi thay thread da nang.

## 2. Prefer Edit Over Retry Messages
- Khi prompt sai hoac can sua huong, dung Edit/Re-edit tin nhan cu neu co the.
- Viec nay loai bo nhanh noi dung sai khoi lich su va giam nhieu nhieu.

## 3. Use Markdown For Fixed Docs
- Chuyen tai lieu co dinh sang `.md` de giam token va tranh loi doc file.
- Dung Project Content cho guideline, brief, style guide, template hay dung lai.

## 4. Keep Inputs Small
- Dua dung file/pham vi can xu ly.
- Neu task lon, chia thanh nhom viec doc lap.
- Khi can review repo, neu muc tieu he, chi ro module, route, component, bug, hoac hanh vi mong muon.

## 5. Model Selection
- Dung model manh cho kien truc, debug kho, refactor lon, code phuc tap.
- Dung model nhanh/nho cho viec ro rang: sua text, format, doc file, thao tac lap lai.

## 6. Working Loop
- Nap principles truoc khi bat dau.
- Doc code va tai lieu lien quan truoc khi sua.
- Sua gon theo pham vi.
- Chay validation.
- Bao cao ket qua ngan gon.
